Pull Request Advice

First and foremost: what is a "Pull Request"?

According to Github, "pull requests let you tell others about changes you've pushed to a branch in a repository on GitHub. Once a pull request is opened, you can discuss and review the potential changes with collaborators and add follow-up commits before your changes are merged into the base branch."

More Information about pull requests can be found here

Tips For Ushering Pull Requests

  • Always make sure your local repository/code is up-to-date with the main branch.
  • Only add files that you edited or that are new and need staging for a push.
  • Make sure your pull request is clear and concise.

An example of an acceptable Pull Request

Don't forget you can use HTML in your Pull Request! It can style up your repsonse, and make all the information clean. Also don't forget to be descriptive. Giving enough information, without giving too much, is the perfect way to inform your fellow developers what you are contributing to the project.
